# CLAUDE.md

This repository uses a handbook-driven workflow.

Before making meaningful edits in this project, read these files in order:
1. `docs/powerstar-handbook.md`
2. `.planning/CURRENT-WORKBOARD.md`

## Required workflow
- Classify every task into one primary line before editing.
- Record Goal / Primary line / Do Not Touch before major changes.
- Keep one main objective per task.
- Do not mix unrelated website, SEO, automation, remotion, and planning changes in one task.
- Update `.planning/CURRENT-WORKBOARD.md` at handoff points so the next session can continue cleanly.

## Project lines
- `website`
- `seo-data`
- `automation`
- `remotion-sidecar`
- `planning-docs`

If the handbook and the current workboard disagree, first update the workboard for current task state, then keep the handbook as the stable source of rules.
