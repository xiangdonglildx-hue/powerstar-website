## Notability Assessment

### Wikipedia Notability Requirements (Companies)

Per Wikipedia guidelines, a company is notable if it has:

1. **Significant coverage** in reliable, independent sources (not self-published)
2. **Depth of coverage** - more than a trivial mention
3. **Multiple sources** - at least 2-3 independent references

## Wikipedia Submission Checklist

### Before Submitting

- [ ] Obtain 2-3 independent press coverage articles
- [ ] Complete Product Hunt launch with positive reception
- [ ] Create Crunchbase/AngelList profiles
- [ ] Ensure at least one tech blog review exists

### Submission Process

1. Create Wikipedia account (not with company email)
2. Build edit history (make small edits to other articles first)
3. Submit draft via Wikipedia:Articles for Creation
4. Wait for reviewer feedback
5. Address any concerns raised by reviewers

### Common Rejection Reasons to Avoid

- **Self-promotion:** Article written by company representative
- **Lack of sources:** No independent reliable sources cited
- **Advertising tone:** Using promotional language
- **COI (Conflict of Interest):** Editor has financial connection to subject

## Alternative: Focus on Other Knowledge Sources

Since Wikipedia is difficult, consider easier alternatives:

### AlternativeTo.net
- Accepts user submissions
- App recommendation platform
- Indexed by search engines and AI

### Fandom Wiki (App-focused)
- Less strict notability requirements
- Can create dedicated wiki pages
- May be indexed by AI crawlers

### GitHub (if code is public)
- Creates indexed profile
- Technical credibility

### Stack Overflow (Answer questions)
- Answer questions about app development topics
- Build reputation as developer
- May reference your apps in relevant answers
