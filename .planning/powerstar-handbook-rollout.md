# PowerStar Handbook Rollout

## Phase 1: Establish the trunk
- Add the handbook.
- Add the current workboard.
- Add repo-local AI instructions.
- Do not move directories yet.

## Phase 2: Start using the reading chain
- Read `docs/powerstar-handbook.md` before any meaningful work.
- Read `.planning/CURRENT-WORKBOARD.md` second.
- Write Goal / Primary line / Do Not Touch before edits.
- Keep one main objective per task.

## Phase 3: Stabilize daily handoff
- Update the workboard at the end of each task.
- Keep In Progress to 1-3 bullets.
- Keep Next Steps to 1-5 bullets.
- Move old completed items into archive or other planning history.

## Phase 4: Review boundary problems
- Identify areas where current files still cross lines.
- Split future cleanup into small `chore/structure-*` tasks.
- Avoid large one-shot reorganization.

## Phase 5: Add automation only if needed
- Consider reminders or hooks after the handbook workflow feels natural.
- Do not add automation that replaces human judgment.
