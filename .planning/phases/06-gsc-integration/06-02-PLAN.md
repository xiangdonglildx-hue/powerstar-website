    Create `scripts/lib/gsc-client.ts` with GSC API wrapper functions.

    Implement three main functions per GSC API endpoints (per CONTEXT.md):

    **1. fetchIndexedPages(siteUrl):**
    Use `urlInspection.index.inspect` API to check indexing status for sitemap URLs.
    - Read sitemap.xml from project root to get list of URLs
    - For each URL, call urlInspection API (limit to 50 URLs per run to avoid quota issues)
    - Count successfully indexed pages
    - Return { indexedCount, totalSubmitted, unindexedUrls }

    **2. fetchTopKeywords(siteUrl, startDate, endDate, limit):**
    Use `searchanalytics.query` API to get search query metrics.
    - Query dimensions: ['query'] for keyword-focused data
    - Metrics: clicks, impressions, ctr, position
    - Sort by clicks descending to get top keywords
    - rowLimit: 50 (for daily check) or 100 (for weekly full)
    - Return array of { query, impressions, clicks, ctr, position }

    **3. fetchSearchMetrics(siteUrl, startDate, endDate):**
    Use `searchanalytics.query` API for aggregate metrics.
    - Query without dimensions for totals
    - Metrics: clicks, impressions, ctr, position
    - Return { totalClicks, totalImpressions, avgCTR, avgPosition }

    **4. fetchCoverageErrors(siteUrl):**
    Use searchanalytics with page dimension to identify potential issues.
    - Query pages with low impressions (< 10) as potential coverage issues
    - Return array of { url, impressions, clicks } for low-performing pages

<task type="auto">
  <name>Task 3: Create main GSC collector script</name>
  <files>scripts/gsc-collector.ts</files>
  <action>
    Create `scripts/gsc-collector.ts` as the main entry point for data collection.

    Script flow:
    1. Load configuration and validate credentials
    2. Calculate date ranges (last 7 days, last 30 days for trends)
    3. Call GSC API functions from gsc-client.ts
    4. Transform and validate data with Zod schemas
    5. Read existing seo-metrics.json to preserve GEO and alerts sections
    6. Merge new GSC data with existing structure
    7. Write updated seo-metrics.json
    8. Create historical snapshot in /data/history/
    9. Clean up history files older than 90 days

    Implementation structure:
    ```typescript
    import { config, validateConfig } from './lib/config.js';
    import { fetchIndexedPages, fetchTopKeywords, fetchSearchMetrics } from './lib/gsc-client.js';
    import { GSCMetricsSchema } from './lib/schemas.js';
    import fs from 'fs-extra';
    import path from 'path';

    async function main() {
      console.log('GSC Collector starting...');
      
      await validateConfig();
      
      // Calculate dates
      const today = new Date();
      const endDate = today.toISOString().split('T')[0];
      const startDate7d = new Date(today - 7 * 24 * 60 * 60 * 1000).toISOString().split('T')[0];
      const startDate30d = new Date(today - 30 * 24 * 60 * 60 * 1000).toISOString().split('T')[0];
      
      // Fetch data
      const indexedPagesData = await fetchIndexedPages(config.gsc.siteUrl);
      const topKeywords = await fetchTopKeywords(config.gsc.siteUrl, startDate30d, endDate, 50);
      const metrics7d = await fetchSearchMetrics(config.gsc.siteUrl, startDate7d, endDate);
      const metrics30d = await fetchSearchMetrics(config.gsc.siteUrl, startDate30d, endDate);
      
      // Build GSC section
      const gscData = {
        indexedPages: indexedPagesData.indexedCount,
        totalSubmitted: indexedPagesData.totalSubmitted,
        topKeywords: topKeywords,
        clicks: {
          last7Days: metrics7d.totalClicks,
          last30Days: metrics30d.totalClicks,
          trend: [], // Will be populated by daily runs
        },
        impressions: {
          last7Days: metrics7d.totalImpressions,
          last30Days: metrics30d.totalImpressions,
          trend: [],
        },
      };
      
      // Validate with Zod
      GSCMetricsSchema.parse(gscData);
      
      // Read existing data and merge
      const existingData = await fs.readJson(config.output.dataPath);
      const updatedData = {
        ...existingData,
        lastUpdated: new Date().toISOString(),
        gsc: gscData,
      };
      
      // Write main file
      await fs.writeJson(config.output.dataPath, updatedData, { spaces: 2 });
      
      // Write historical snapshot
      const historyFile = path.join(config.output.historyPath, `gsc-${endDate}.json`);
      await fs.writeJson(historyFile, { lastUpdated: endDate, gsc: gscData }, { spaces: 2 });
      
      // Cleanup old history files (90 days retention)
      await cleanupHistory();
      
      console.log('GSC Collector completed successfully');
    }

    async function cleanupHistory() {
      const files = await fs.readdir(config.output.historyPath);
      const cutoff = new Date(Date.now() - config.retentionDays * 24 * 60 * 60 * 1000);
      
      for (const file of files) {
        if (file.startsWith('gsc-') && file.endsWith('.json')) {
          const dateStr = file.replace('gsc-', '').replace('.json', '');
          const fileDate = new Date(dateStr);
          if (fileDate < cutoff) {
            await fs.remove(path.join(config.output.historyPath, file));
            console.log(`Removed old history file: ${file}`);
          }
        }
      }
    }

    main().catch(err => {
      console.error('GSC Collector failed:', err.message);
      process.exit(1);
    });
    ```

    Add error handling for:
    - Authentication failures (exit with clear message)
    - API quota exceeded (log and exit gracefully)
    - Network errors (retry up to 3 times)
    - Invalid responses (skip and log warning)
