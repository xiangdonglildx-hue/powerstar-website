<task type="auto">
  <name>Task 4: Create Service Account setup documentation</name>
  <files>scripts/GCP-SA-SETUP.md</files>
  <action>
    Create comprehensive Service Account setup documentation at `scripts/GCP-SA-SETUP.md`.

    The document must include step-by-step instructions for:

    ## Section 1: Create Google Cloud Project (if needed)
    - Go to console.cloud.google.com
    - Create new project or select existing
    - Note the project ID

    ## Section 2: Enable Google Search Console API
    - Navigate to "APIs & Services" > "Library"
    - Search for "Google Search Console API"
    - Click "Enable"
    - Wait for activation (1-2 minutes)

    ## Section 3: Create Service Account
    - Navigate to "IAM & Admin" > "Service Accounts"
    - Click "Create Service Account"
    - Name: "powerstar-gsc-collector"
    - Description: "Service account for SEO monitoring scripts"
    - Click "Create and Continue"
    - Skip role assignment (not needed for API access)
    - Click "Done"

    ## Section 4: Generate JSON Key File
    - Click on the created Service Account
    - Go to "Keys" tab
    - Click "Add Key" > "Create new key"
    - Select "JSON" format
    - Download and save as `scripts/credentials.json`
    - IMPORTANT: Never commit this file to git

    ## Section 5: Add Service Account to Search Console
    - Go to search.google.com/search-console
    - Select the powerstarapps.com property
    - Go to "Settings" > "Users and permissions"
    - Click "Add user"
    - Enter the Service Account email (found in credentials.json as "client_email")
    - Permission: "Full" or "Restricted" (Restricted with "View" permission is sufficient)
    - Click "Add"

    ## Section 6: Verify Setup
    - Copy .env.example to .env
    - Run: cd scripts && npm install && npx tsx test-auth.ts
    - Expected: "Authentication successful" message

    Include troubleshooting section for common errors:
    - 403 Permission Denied: Service Account not added to GSC property
    - 404 Not Found: Wrong GSC_SITE_URL (use exact property URL including sc-domain: prefix)
    - Invalid credentials: JSON key file path incorrect

    Reference STACK.md and CONTEXT.md for authentication approach (Service Account).
  </action>
